Sharing mode

By default this bridge keeps your data on your network. Sharing happens two ways:

  • Per-publish: pass share_with when you publish — the bridge encrypts to those specific recipients.
  • Per-bridge: set EXTERNAL_RELAYS=wss://relay.example,... in your bridge env to mirror events to public Nostr relays.

There's no global on/off switch. Sharing is a gradient — choose per-thing.

How devices connect

Point any MCP client at http://<your-host>:<port>/mcp as a custom connector.

Phone (claude.ai mobile / ChatGPT app) Add bridge URL as custom MCP connector. Use a stable hostname (Tailscale or LAN mDNS) so it survives across networks.
Desktop (Claude Code, Cursor, custom) Same URL. Or run the bridge on the same machine and use stdio.
Sharing with another person Per-publish: their bridge needs to reach the same relay. For now, set EXTERNAL_RELAYS on both bridges to a common public Nostr relay.
Open source. Local-first by design. No hosted server we operate is required for personal use.
