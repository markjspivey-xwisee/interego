abac: — ABAC evaluation pattern abac:

Namespace https://markjspivey-xwisee.github.io/interego/ns/abac# · 15 terms.

L2 pattern ontology for attribute-based access-control evaluation on top of iep:AccessControlPolicy descriptors. Specifies evaluator, attribute resolver, decision cache, and audit record as constructions over L1 primitives.

Machine-readable form: abac.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. abac:Allowed = https://markjspivey-xwisee.github.io/interego/ns/abac#Allowed).

Terms

abac:Allowed Individual
Evaluation result: the governed action MAY proceed. Possibly with duties.
abac:AttributeResolver Class
Builds the subject's attribute graph by aggregating facets from: the subject's own descriptor(s), peer-issued attestations cited via provenance, and any trust attestations (e.g. amta:TrustAttestation) reachable from the subject. A natural transformation from a subject IRI to an attribute graph, grounded in the facet lattice.
abac:cacheValidUntil Datatype property
Upper bound on when this decision's cached form remains valid. After this instant, re-evaluation is required.
abac:decisionAt Datatype property
Instant of the decision. Enables cache validity reasoning.
abac:DecisionCache Class
Short-circuits repeat evaluations. A cached decision is stored as a iep:TrustFacet-shaped attestation — the issuer is the evaluator, the validity window is explicit, and the attestation is verifiable. A stale cache entry therefore becomes a verifiable stale attestation rather than silent drift.
abac:Denied Individual
Evaluation result: the governed action MUST NOT proceed.
abac:evaluatedPolicy Object property
The iep:AccessControlPolicy that an evaluation record evaluated.
abac:EvaluationRecord Class
An auditable record of one policy evaluation: inputs, decision, duties, matched policies, timestamp. As a iep:ContextDescriptor it composes with the rest of the system (you can publish it, cite it, validate it, query it) — the audit trail IS linked data.
abac:Evaluator Class
A stateless function that maps (policy, subject, resource, environment) to a decision. Constructed from a policy descriptor + a context — both L1 primitives — so this class is the pattern-level naming of a computation grounded in L1.
abac:Indeterminate Individual
Evaluation result: insufficient attribute information to decide. Under the open-world assumption at the federation layer, indeterminate is distinct from denied.
abac:matchedRule Object property
A policy rule that matched during evaluation; used in composed (multi-policy) decisions for traceability.
abac:Policy Class
The pattern-level name for an attribute-based access-control policy. Structurally identical to the L1 iep:AccessControlPolicy (a first-class context descriptor carrying a SHACL predicate, a governed action, a deontic mode, and optional duties); declared here so verticals can type a policy graph in the abac: vocabulary while remaining a iep:AccessControlPolicy for every L1 reader.
abac:PolicyContext Class
The ambient state for one policy evaluation: the subject's attribute graph, the resource being accessed, the environmental facets (time, federation origin, etc.), and any pre-resolved attestations. Constructed from the seven L1 facets of the invoking request.
abac:resolvedAttributes Object property
The attribute graph resolved for one evaluation, as a set of facet IRIs.
abac:verdict Object property
The decision: abac:Allowed, abac:Denied, or abac:Indeterminate.
