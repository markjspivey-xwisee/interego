PGSL — Poly-Granular Sequence Lattice 1.0 Ontology pgsl:

Namespace https://markjspivey-xwisee.github.io/interego/ns/pgsl# · version 1.0 · 39 terms.

Content-addressed, lattice-structured representation of text where every fragment gets a stable atom identity and compositional relationships form a bounded lattice. Defines atoms (level 0), fragments (level ≥ 1), pullback squares for the overlapping pair construction, constituent morphisms, transitive containment, and the co-occurrence matrix that grounds usage-based semantics. Aligned with W3C PROV-O for full provenance and designed to compose with the Interego (iep:) typed context layer above it.

Machine-readable form: pgsl.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. pgsl:AcyclicityShape = https://markjspivey-xwisee.github.io/interego/ns/pgsl#AcyclicityShape).

Terms

pgsl:AcyclicityShape Shape
Validates the lattice is a DAG (no fragment is its own ancestor).
pgsl:apex Object property
The apex fragment of a pullback square — the fragment that simultaneously contains both the left and right constituents and equals their categorical pullback over the shared overlap. Functional.
pgsl:Atom Class
A leaf node in the lattice containing a single primitive value (string, number, or boolean). Atoms have level 0 by definition and are the minimum granularity unit. Two atoms are equal if and only if their values are content-equal — atom identity is canonical and content-addressed.
pgsl:atomCount Datatype property
The total number of atoms (level-0 nodes) in a lattice instance. Used for retrieval scoring and SPARQL summary queries.
pgsl:Atoms Class
A collection of pgsl:Atom nodes minted together (e.g. one content-addressed atom per launchable lesson). The plural envelope over pgsl:Atom membership, used when a result reports the set of atoms produced by an operation; not a new lattice node kind.
pgsl:AtomShape Shape
pgsl:attributedTo Object property
The entity (agent / user / organization) a node is attributed to. Analogue of prov:wasAttributedTo scoped to the pgsl: namespace.
pgsl:CanonicalityShape Shape
Validates URIs are content-addressed (same plaintext -> same URI within a pod's recipient set).
pgsl:ConstituentIntegrityShape Shape
Validates that fragment constituents are consistent (left/right pullback recoverable).
pgsl:ConstituentMorphism Class
An arrow in the lattice category: a structure-preserving inclusion of one node into a fragment that contains it. Constituent morphisms compose to form the transitive containment relation and witness the lattice's order structure.
pgsl:constituentOf Object property
Inverse of pgsl:item. The subject node is a direct constituent of the target fragment. Inferable; usually not asserted explicitly.
pgsl:containedIn Object property
Inverse of pgsl:contains.
pgsl:contains Object property
Transitive containment: this fragment contains the target node either directly or via a chain of intermediate fragments. Equivalent to the reflexive-transitive closure of pgsl:item over a fragment subtree.
pgsl:Fragment Class
A composite node spanning multiple atoms or sub-fragments. Fragments have level ≥ 1, where the level equals the number of base atoms transitively spanned. A fragment is content-addressed by the canonical CID of its ordered item list, so two fragments with the same items in the same order are the same fragment.
pgsl:fragmentCount Datatype property
The total number of fragments (level ≥ 1 nodes) in a lattice instance.
pgsl:FragmentShape Shape
pgsl:generatedAt Datatype property
ISO 8601 timestamp recording when a pgsl: node was minted.
pgsl:hasAtom Object property
Links a descriptor or record to the content-addressed pgsl:Atom holding its payload — the lattice anchor for the resource's content. Lets a reader recover the immutable, content-addressed payload behind a mutable descriptor.
pgsl:height Datatype property
The topological depth of a fragment from the lattice root. Height is distinct from level — two fragments may share a level but have different heights if one was constructed by deeper nested compositions. Functional.
pgsl:item Object property
Direct containment: the target node is one of this fragment's immediate constituents. For SPARQL queryability, the items are also exposed individually in addition to the canonical RDF list (pgsl:itemList).
pgsl:itemList Object property
The ordered sequence of constituent nodes, encoded as an RDF list (rdf:first / rdf:rest / rdf:nil). Order is significant — two fragments with the same items in different orders are distinct fragments with distinct CIDs.
pgsl:Lattice Class
A complete instance of a Poly-Granular Sequence Lattice — the closure of all nodes under the lattice operations (meet, join, pullback). A Lattice resource exposes summary statistics (atom count, fragment count, max level) and serves as a navigable root for SPARQL or HATEOAS access.
pgsl:leftConstituent Object property
The left fragment in the overlapping pair construction. Functional: a pullback square has exactly one left constituent.
pgsl:level Datatype property
The granularity level of a node, equal to the number of base atoms it transitively spans. Level 0 ⇔ atom; level ≥ 1 ⇔ fragment. Functional: each node has exactly one level.
pgsl:LevelConsistencyShape Shape
Validates that fragment level = max(left.level, right.level) + 1.
pgsl:maxLevel Datatype property
The maximum level reached by any fragment in a lattice instance — equivalently, the size of the longest contained sequence.
pgsl:metagraph Object property
Reference to a named graph holding meta-level annotations about this pgsl: node.
pgsl:Node Class
Abstract supertype for any node in the Poly-Granular Sequence Lattice. Every node has a content-addressed URI, a level (granularity), and provenance attribution. Atoms and Fragments are the two concrete subtypes; they are mutually disjoint and partition the set of nodes.
pgsl:overlap Object property
The shared sub-node (pullback fiber product) of the overlapping pair. The overlap is what makes the pullback square commute: both the left and right constituents project down onto it via constituent morphisms. Functional.
pgsl:OverlapValidityShape Shape
Validates pullback overlap is non-empty + canonically minimal.
pgsl:PullbackSquare Class
The categorical pullback encoding the overlapping pair construction: given two fragments that share a common sub-fragment, the pullback square (apex, leftConstituent, rightConstituent, overlap) commutes — both projections from the apex equal the inclusions into the overlap. This is the formal mechanism by which PGSL captures sequence overlap without duplication.
pgsl:PullbackSquareShape Shape
pgsl:resolved Datatype property
The resolved text representation of a fragment (full reconstructed string across all constituents).
pgsl:rightConstituent Object property
The right fragment in the overlapping pair construction. Functional: a pullback square has exactly one right constituent.
pgsl:sign Object property
Peircean sign associated with this node — the representamen aspect of the triadic sign/object/interpretant structure.
pgsl:source Object property
The source (domain) of a constituent morphism — the smaller node that is being included into a containing fragment.
pgsl:sourceFormat Datatype property
The original format the content was extracted from before being atomized into the lattice (e.g. 'scorm', 'pdf', 'html', 'cmi5'). Recorded on the minting record for provenance.
pgsl:target Object property
The target (codomain) of a constituent morphism — the containing fragment that the source is being included into.
pgsl:value Datatype property
The primitive value of an atom. Functional: each atom has exactly one value. The value type may be xsd:string, xsd:integer, xsd:double, or xsd:boolean depending on the original datum.
