WKS — Weak-Signal vocabulary wks:

Namespace https://markjspivey-xwisee.github.io/interego/ns/wks# · 11 terms.

Typed weak signals for complexity-aware sensemaking. Aggregation is structural (SPARQL through wks:about); reinforcement is automatic; dispositions are candidate explanations, not proofs.

Machine-readable form: wks.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. wks:about = https://markjspivey-xwisee.github.io/interego/ns/wks#about).

Terms

wks:about Object property
The phenomenon, system, situation, or other descriptor the weak signal points at. The structural matching key for cross-observer reinforcement: two wks:WeakSignal descriptors with the same wks:about value count as independent observations of the same potential disposition.
wks:candidateDisposition Object property
Optional link to a sat:Disposition that, if the signal is real, the system might be exhibiting. NOT a claim that the disposition is present — a candidate explanation worth probing. Pair with iep:modalStatus = Hypothetical.
wks:hasNarrativeEvidence Object property
Links the weak signal to one or more micro-narratives that motivate it. Multiple cardinality permitted — an observer may anchor a single signal in several distinct observations.
wks:NarrativeEvidence Class
A free-form micro-narrative that anchors a wks:WeakSignal. Not a fact-claim — a description of what was observed in observer's own words. Stored as content-addressed pgsl:Atom so two observers producing identical narratives collide on IRI (structural reinforcement at the description layer).
wks:observedBy Object property
The agent or human who observed the signal. Required for cross-observer reinforcement counting (multiple signals from one observer count as one).
wks:Reinforcement Class
A computed measure of how many independent observers have published wks:WeakSignal descriptors with the same wks:about subject. A summary descriptor that supersedes-chains older counts as new signals arrive. NOT an aggregation of opinions — a count of independent observations.
wks:reinforcementCount Datatype property
Number of independent observers (distinct wks:observedBy values) who published wks:WeakSignal descriptors with the same wks:about. A reinforcement of 1 is just an observation; ≥2 is a candidate weak signal worth probing; ≥N (per the slice's Cynefin domain) may justify amplification.
wks:reinforcementWindow Datatype property
ISO-8601 duration over which the reinforcement count was computed. Required to interpret the count: 5 observers in an hour means something different from 5 in a year.
wks:reinforces Object property
Optional explicit link from one weak signal to another it reinforces. Usually unnecessary — two signals with the same wks:about subject reinforce each other structurally. Use only when an observer wants to attribute interpretation lineage.
wks:reinforcesAbout Object property
The wks:about subject this reinforcement count was computed for.
wks:WeakSignal Class
A subtle observation that, on its own, does not justify a strong claim — but in aggregate across multiple independent observers, surfaces a latent disposition. Always Hypothetical (the observer asserts WAS-OBSERVED, not WHAT-IT-MEANS). Subtype of iep:ContextDescriptor; the descriptor's modal status MUST be Hypothetical.
