Interego 1.0 — Interrogatives Core Ontology ie:

Namespace https://markjspivey-xwisee.github.io/interego/ns/interego# · version 1.0 · 34 terms.

The core ontology of Interego, the improvisational signification platform. Defines the eleven canonical interrogatives (Who, What, Where, When, Why, How, Which, WhatKind, HowMuch, Whose, Whether) as a SKOS concept scheme; the act of posing an interrogative and its response (both aligned with PROV-O); the Peircean semiotic triad of Sign, Object, and Interpretant; and the unit of emergent meaning (Signification). Each existing technical layer of Interego — the substrate, the typed-context descriptors, and the agent harness — answers specific interrogatives. This ontology provides the user-facing grammar over that technical machinery.

Machine-readable form: interego.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. ie:about = https://markjspivey-xwisee.github.io/interego/ns/interego#about).

Terms

ie:about Object property
The focal entity or entities the interrogative is about. Typically a pgsl:Node (substrate data) or iep:ContextDescriptor (typed context). Gives the act a concrete target; distinct from the interrogative TYPE which is abstract.
ie:Act Class
A speech act of posing an interrogative. Records who asked (posedBy), optionally who they asked (posedTo), what kind of interrogative (interrogativeType), and what the question is about (about). Modeled as a prov:Activity so PROV tooling can walk dialog lineage and so policy decisions, traces, and evaluations can all be attached uniformly.
ie:answeredBy Object property
Links an Act to a Response that addresses it. Multi-valued — an act can have many responses from different agents, or the same agent can offer refinements over time.
ie:confidence Datatype property
Confidence in [0.0, 1.0] associated with a response. Typically computed by ieh:RuntimeEval and carried over to the response. Functional — each response has one confidence value.
ie:derivesFrom Object property
Links an interpretant back to the sign it interprets. Functional — each interpretant interprets exactly one primary sign (though chains of interpretants can interpret each other, forming 'semiotic chains' in Peirce's sense).
ie:emergesFrom Object property
The acts and/or responses this signification retroactively binds together into a coherent meaning unit. Multi-valued — a signification typically emerges from multiple prior exchanges.
ie:forAgent Object property
The agent whose interpretant this is. Functional — each interpretant belongs to exactly one agent, because meaning is agent-relative. To represent a shared interpretation across agents, use multiple interpretant resources that reference the same content.
ie:grounds Object property
The substrate data this response draws upon. Typically pgsl:Node instances (atoms or fragments). Subproperty of prov:used so the response's inputs are discoverable via standard PROV walks. A response with no ie:grounds is a pure claim; a response with grounds is traceable back to primary data.
ie:How Concept
Methodical/procedural interrogative. Asks about means, method, or manner — the procedure by which something came to be or the way it is structured. Answered by iep:ProvenanceFacet (how it was produced), pgsl:PullbackSquare (how it is composed), and ieh:strategyUsed (how the harness approached a question).
ie:HowMuch Concept
Quantitative interrogative. Asks about quantity, extent, magnitude, or degree. Includes both continuous (how much water) and discrete (how many items) variants. Answered by pgsl:level / pgsl:atomCount / pgsl:fragmentCount (substrate cardinality) and by ieh:confidence (degree of certainty).
ie:improvisesOn Object property
A signification may IMPROVISE on a prior signification — extending it in a direction not originally planned, much as a jazz musician improvises on a theme. The relationship is looser than rdfs:subClassOf or ie:refines — it captures the emergent, responsive, unscripted nature of real-time meaning construction. This property is what makes Interego an 'improvisational' platform rather than merely a dialog logger.
ie:Interpretant Class
The meaning a particular agent derives from a particular sign. CRITICAL: interpretants are agent-relative. Two agents reading the same sign may produce different interpretants — this is not an error but the fundamental structure of intersubjective meaning. Interego records interpretants as first-class resources so disagreements can be compared, reconciled, or preserved as productive ambiguity.
ie:interpretedBy Object property
The interpretants a sign gives rise to. Multi-valued — a single sign can have many interpretants, one per agent (or per agent-context pair). This is where intersubjective disagreement lives.
ie:Interrogative Class
The abstract parent for all interrogative types. Also a SKOS concept scheme — the eleven canonical interrogatives are skos:Concept instances of this class. An Interrogative is not a question in natural language; it is the TYPE of question, the semantic category that a given question falls into. Real questions posed by agents are modeled as ie:Act instances whose ie:interrogativeType is one of these types.
ie:interrogativeType Object property
The kind(s) of interrogative this act poses. Multi-valued because a single natural-language question often combines multiple interrogatives (e.g. 'when did Alice tell me about it?' combines Who, What, and When).
ie:Object Class
The thing a sign is about — what Peirce called the 'dynamic object'. Distinct from the sign itself, and distinct from any particular agent's interpretation. In practice, objects are often pgsl:Node instances (substrate data) or iep:ContextDescriptor references.
ie:posedBy Object property
The agent who posed this interrogative act. Functional — each act has exactly one asker. Subproperty of prov:wasAssociatedWith so PROV tooling can find the asker via standard queries.
ie:posedTo Object property
The agent or group the interrogative is directed at. Optional — absent when the question is posed to no one in particular (e.g. a monologue or a rhetorical question). Multi-valued — a question can be posed to multiple recipients.
ie:refines Object property
This act narrows or specifies a prior act. The refining act is always more specific than the one it refines — the interrogative type may stay the same but the scope contracts (e.g. 'what did Alice say?' refines 'who said what?'). Forms a directed acyclic graph of dialog evolution.
ie:reframes Object property
This act changes the interpretive frame of a prior act — swaps paradigms, replaces presuppositions, or shifts the semantic register. A reframe is a more aggressive move than a refinement: the relationship between the old and new question is not one of containment but of perspective change. In Saussurean terms, a reframe changes the paradigmatic axis.
ie:Response Class
Content offered in response to an Interrogative Act. Grounded in substrate data (ie:grounds), attributed to a responding agent (prov:wasAttributedTo), and optionally evaluated for confidence (ie:confidence). A Response is never final — it can itself provoke further Acts that refine or reframe the original question.
ie:responseTo Object property
Links a Response back to the Act it addresses. Functional — a single response responds to exactly one primary act. If a response happens to also address other acts, record that via additional ie:answers triples.
ie:Sign Class
An interpretable token — whatever an agent takes to stand for something else. Peircean signs are NOT linguistic only; they include gestures, data patterns, sensor readings, and (recursively) other signs. In Interego, pgsl: atoms and fragments are frequently treated as signs when an agent interprets them.
ie:Signification Class
A unit of co-constructed meaning emerging from an interrogative exchange. Signification is NOT a pre-planned assertion — it is what participants retroactively recognize as having been jointly constructed. A signification emerges from one or more ie:Act / ie:Response exchanges (ie:emergesFrom), may improvise on prior significations (ie:improvisesOn), and is grounded in substrate data through its constituent responses. Significations are the atoms of meaning in Interego's improvisational signification platform.
ie:signifies Object property
The primary sign-to-object relation. Reads: 'this sign signifies that object.' The relation is mediated by interpretants (see ie:interpretedBy) but for convenience the direct sign→object link is also recorded.
ie:What Concept
Entity/object interrogative. Asks about the referent of a sign — what thing is being discussed. Answered by pgsl:Atom (atomic value) and pgsl:Fragment (composite structure) in the substrate layer.
ie:WhatKind Concept
Typological interrogative. Asks about the type, category, or class of a thing. Answered by iep:SemioticFacet (under what interpretive frame?) and by rdf:type / skos:Concept membership.
ie:When Concept
Temporal interrogative. Asks about time — instants, intervals, durations, and temporal relations. Answered by iep:TemporalFacet of context descriptors, aligned with OWL-Time.
ie:Where Concept
Spatial/locational interrogative. Asks about location, container, or place — including non-geographic containers like pods, graphs, or organizational boundaries. Answered by iep:FederationFacet (where is this hosted?) and by container relationships in the substrate.
ie:Whether Concept
Modal interrogative. Asks about truth, possibility, permission, or likelihood. Encompasses yes/no questions, epistemic modality (is it true?), deontic modality (is it permitted?), and alethic modality (is it possible?). Answered by iep:TrustFacet (how certain?), ieh:PolicyDecision (is it permitted?), and ieh:RuntimeEval (is the answer reliable?).
ie:Which Concept
Selection/paradigmatic interrogative. Asks for a choice from a known set of alternatives — the Saussurean paradigm axis. Distinct from ie:What (which asks open-endedly) in that ie:Which assumes a closed set of options. Answered by ieh:Decision (which action to take) and by iep:restriction (which facets to include in a composition).
ie:Who Concept
Agent/subject interrogative. Asks about the identity of an agent involved in an event, proposition, or assertion. Answered by iep:AgentFacet in the typed-context layer and by prov:Agent in provenance traces.
ie:Whose Concept
Possessive/ownership interrogative. Asks about control, stewardship, or possession — who has the authority to modify, access, or speak for something. Answered by iep:AccessControlFacet (access rights) and iep:AgentFacet onBehalfOf delegation.
ie:Why Concept
Causal/motivational interrogative. Asks about reasons — both proximate causes (the mechanism that produced an outcome) and motivating reasons (the goal or intention behind an action). Answered by iep:ProvenanceFacet (causal provenance) and by ieh:Decision justification (motivating reasons).
