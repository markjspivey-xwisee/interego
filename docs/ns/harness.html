Interego Harness 1.0 Ontology cgh:

Namespace https://markjspivey-xwisee.github.io/interego/ns/harness# · version 1.0 · 158 terms.

The harness ontology formalizes the agent framework, policy engine, runtime evaluation, decision functor, and affordance decorator machinery that wraps an LLM and reasons over the PGSL substrate. Each harness operation produces a PROV-O trace; each affordance is exposed as a Hydra Operation; each policy decision is grounded in deontic rules aligned with ODRL. Together with iep: and pgsl:, this ontology provides the third layer of a co-designed substrate-and-harness system.

Machine-readable form: harness.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. cgh:AATRegistry = https://markjspivey-xwisee.github.io/interego/ns/harness#AATRegistry).

Terms

cgh:AATRegistry Class
A registry mapping AAT identifiers to their AbstractAgentType definitions. Used by the harness to resolve an agent's type at runtime.
cgh:Abstain Concept
The question cannot be answered from available evidence. Return an abstention.
cgh:AbstainStrategy Concept
Take no action. Used when no affordance has sufficient grounding.
cgh:AbstractAgentType Class
A typed bundle of capabilities and deontic constraints that classifies an agent. AATs determine what the agent CAN perceive (canPerceive), what it CAN do (canAct), and what invariants it MUST preserve (mustPreserve). An agent's effective capabilities are the conjunction of its AAT's permissions and any further restrictions imposed by the policy engine. Modeled as a prov:Plan because an AAT specifies the procedure an agent follows.
cgh:Accept Concept
Use the answer as-is. Confidence is high enough to commit.
cgh:action Datatype property
The action a policy rule governs (e.g. 'mintAtom', 'publishContext'). Use '*' for any action.
cgh:Affordance Class
A capability the harness can offer an agent: an action with preconditions and effects, targeting a specific node. Affordances are the harness's vocabulary for 'what can I do next?'. Aligned with hydra:Operation for HATEOAS interoperability.
cgh:AffordanceDecorator Class
A pluggable component that, given a DecoratorContext (a node and its surrounding lattice state), produces a set of DecoratedAffordances and optional StructuralSuggestions. Decorators are how the harness exposes capabilities to agents in a way that respects domain trust levels.
cgh:affordanceTarget Object property
The lattice node this affordance acts upon. Functional.
cgh:affordanceType Datatype property
The kind of action this affordance offers (e.g. 'mintAtom', 'verifyCoherence', 'navigate').
cgh:agentAAT Object property
The AAT under which the agent was acting when the trace was produced. Functional — an agent acts under exactly one AAT per operation.
cgh:AgentAction Class
A typed audit record for a tool call an agent runtime emitted on behalf of an owner. Subject of the descriptor; complements (does not replace) prov:Activity. Used by the integrations layer (e.g. @interego/compliance-overlay) so SPARQL queries can filter down to agent-action audit entries. The cited regulatory controls go via dct:conformsTo into the framework ontologies (eu-ai-act:, nist-rmf:, soc2:) — no new compliance vocab is minted here.
cgh:AgentMemory Class
Memory layer sizes for an agent: semantic (facts), episodic (events), procedural (skills). Used by the harness to budget retrieval and decay.
cgh:allowed Datatype property
Whether a PolicyDecision allowed the action. Functional.
cgh:appliesToToolType Object property
Optional scope IRI — if present, the constraint only applies to tools of this type. If absent, the constraint applies to ALL promotion attempts.
cgh:appNarrative Datatype property
Human-readable description of what the app does and how to use it — equivalent in role to a README. Renders alongside the typed structure for human reviewers; the typed structure is what consumer agents walk.
cgh:Away Concept
Agent is connected but idle.
cgh:bundlesAffordance Object property
Lists an affordance descriptor included in this protocol version. Multiple cardinality permitted — every affordance the protocol guarantees is referenced exactly once.
cgh:Busy Concept
Agent is online but should not be interrupted.
cgh:canAct Datatype property
An action tag this AAT is permitted to perform (e.g. 'mintAtom', 'publishContext', 'verifyCoherence'). Multi-valued.
cgh:canPerceive Datatype property
A capability tag this AAT is permitted to read or observe (e.g. 'pgsl:Atom', 'iep:TrustFacet'). Multi-valued. Capabilities are intersected with policy permissions at runtime.
cgh:causalChain Object property
Prior decisions this decision depends on. Forms a DAG of decision lineage.
cgh:coherenceBasis Object property
Other agents' observations that corroborate this decision.
cgh:CommunityTrust Concept
Decorator from the broader community. Affordances may be filtered out for sensitive operations.
cgh:composes Object property
Affordances the app composes, in calling order. Use rdf:Seq or an RDF list when order matters; multiple cardinality otherwise. The same affordance may appear multiple times if the app calls it more than once.
cgh:confidence Datatype property
Numeric confidence score in [0.0, 1.0] derived from structural signals. Functional.
cgh:ConfidenceLevel Class
A SKOS concept scheme of qualitative confidence levels derived from the [0,1] confidence score: high (≥ 0.8), medium (≥ 0.5), low (≥ 0.3), uncertain (< 0.3).
cgh:conflictsWith Object property
Another AAT this AAT cannot run concurrently with — e.g. an Executor and an Arbiter cannot both act on the same enclave at the same time. Symmetric in spirit but asserted in both directions.
cgh:Constrain Concept
Add a SHACL or coherence constraint to restrict valid lattice configurations.
cgh:Conversation Class
An ordered collection of conversation messages between participants. Modeled as a prov:Collection so PROV tooling can walk message lineage.
cgh:ConversationMessage Class
A single message in a conversation: sender, content, timestamp, and optional link to the PGSL chain that materialized the message.
cgh:CountingQuestion Concept
Question of the form 'how many X' where X is a discrete enumerable category.
cgh:coverage Datatype property
Fraction of observations that resulted in a decision, in [0.0, 1.0].
cgh:datesExplicit Datatype property
For temporal questions: whether the relevant dates were explicitly stated (true) or inferred (false).
cgh:Decision Class
A single resolved (affordance, confidence, justification) tuple. Decisions reference their causal chain of prior decisions and the coherence basis (which other agents' observations corroborate the decision).
cgh:DecisionResult Class
The output of the decision functor: a chosen strategy, the set of decisions produced, the coverage fraction (what % of observations were grounded in a decision), and the list of ungrounded observations that need further attention.
cgh:DecisionStrategy Class
A SKOS concept scheme of the four strategies the decision functor can choose: exploit, explore, delegate, abstain.
cgh:DecoratedAffordance Class
An Affordance produced by a specific decorator, carrying the decorator's id, name, trust level, confidence, and rationale. The trustLevel allows agents to filter affordances by provenance — e.g. accept only system-trusted actions.
cgh:DecoratorContext Class
The input passed to a decorator: the target node, its kind/level/value, the items it contains (for fragments), the resolved source/target options for navigation, applicable constraints, the node's containers, and the existing affordances produced by earlier decorators in the registry.
cgh:decoratorId Datatype property
Stable identifier of the decorator that produced this affordance or suggestion.
cgh:decoratorName Datatype property
Human-readable name of the decorator.
cgh:DecoratorRegistry Class
An ordered set of decorators applied to each node in priority order. Earlier decorators (lower priority numbers) run first; later decorators see the affordances already produced and can augment or override.
cgh:DecoratorResult Class
The output of running one decorator: a list of DecoratedAffordances and an optional list of StructuralSuggestions.
cgh:Delegate Concept
Hand off the decision to a higher-trust agent (typically an Arbiter). Used when the current agent's AAT does not permit the required action.
cgh:Deny Concept
The rule explicitly prohibits the action when the condition holds. Aligned with odrl:prohibition.
cgh:DeonticMode Class
A SKOS concept scheme of the three deontic modes a policy rule can take: permit, deny, duty. Aligned with ODRL.
cgh:domain Datatype property
The domain a decorator targets (e.g. 'core', 'xapi', 'persistence', 'federation').
cgh:durationMs Datatype property
Action wall-clock duration in milliseconds. Useful for SLA / latency audit queries; finer-grained than prov:startedAtTime / prov:endedAtTime when sub-second resolution matters and clock skew across pods would otherwise mislead.
cgh:duty Datatype property
An obligation imposed by the matched policy rules. Multi-valued.
cgh:Duty Concept
The rule imposes an obligation that must be discharged when the action is performed. Aligned with odrl:duty.
cgh:effect Datatype property
A state change this affordance produces if executed. Multi-valued; conjunctive.
cgh:episodicSize Datatype property
Number of episodic-memory entries (events) the agent currently holds.
cgh:errorMessage Datatype property
Failure reason if ieh:success is false. Functional.
cgh:Escalate Concept
Confidence is too low to act autonomously. Request human review or hand off to a higher-trust agent.
cgh:EvalAction Class
A SKOS concept scheme of the four actions a runtime evaluation can recommend: accept, retry, escalate, abstain.
cgh:EvalHistoryEntry Class
A single record of a runtime evaluation with optional ground-truth correctness. Used to compute historical accuracy per (questionType, strategy) pair.
cgh:eventCount Datatype property
Number of revision events observed in the window (numerator of the rate).
cgh:ExpertTrust Concept
Decorator written by a vetted domain expert. Affordances are presented but may be flagged for review.
cgh:explanation Datatype property
Human-readable explanation of the confidence assessment.
cgh:Exploit Concept
Take the highest-confidence affordance from the current observations. Optimizes for known-good action.
cgh:Explore Concept
Take an affordance that probes uncertain regions of the lattice. Optimizes for information gain.
cgh:extractedItemCount Datatype property
For counting questions: how many items the panel extracted.
cgh:FactualQuestion Concept
Direct factual lookup question with no aggregation or temporal reasoning.
cgh:Group Concept
Combine multiple sibling nodes under a new parent fragment.
cgh:hasAffordance Object property
Links a Decision to the affordance it selects.
cgh:hasConversation Object property
Links a PersonalBroker to its conversations.
cgh:hasMessage Object property
Links a Conversation to its messages.
cgh:hasRule Object property
Links a PolicyEngine to one of its rules. Order is significant — earlier rules can preempt later ones.
cgh:HighConfidence Concept
Confidence score ≥ 0.8. The harness recommends accepting the answer without retry.
cgh:isAbstention Datatype property
Whether the answer was an abstention. Affects how the action recommendation is computed.
cgh:isAdjacentPossibleTo Object property
Asserts that the source descriptor's situation could plausibly transition to the target descriptor's situation under known constraints, without claiming the transition will happen. The honest framing for complex domains: hold the space of next-move possibilities open without committing to a prediction. Use ieh:reachabilityRationale to record why the transition is reachable; iep:modalStatus on the descriptor carrying this property SHOULD be Hypothetical.
cgh:justification Datatype property
Human-readable justification for the decision.
cgh:KnowledgeUpdateQuestion Concept
Question whose answer requires reconciling later sessions overriding earlier ones.
cgh:level Object property
Qualitative confidence level derived from ieh:confidence by thresholding.
cgh:LLMTrust Concept
Affordance generated by an LLM advisor decorator. Lowest default trust — agents should verify before acting.
cgh:LowConfidence Concept
Confidence score ≥ 0.3 and < 0.5. The harness recommends retrying with an alternative strategy.
cgh:matchedRule Object property
A policy rule that fired during this decision.
cgh:maxConcurrent Datatype property
Maximum number of agents of this AAT that may operate concurrently in a single enclave. Used by the harness scheduler to enforce parallelization limits.
cgh:measuresFederationSlice Object property
The iep:Federation slice (or descriptor identifying the slice) being measured. Lets multiple realignment measures coexist for different slices of the same pod.
cgh:MediumConfidence Concept
Confidence score ≥ 0.5 and < 0.8. The harness may accept the answer or retry with a different strategy depending on configuration.
cgh:mode Object property
The deontic mode of a policy rule.
cgh:mustPreserve Datatype property
An invariant tag this AAT is obliged to preserve across all of its actions (e.g. 'lattice-coherence', 'provenance-chain'). Violations of mustPreserve invariants trigger policy denials regardless of canAct permissions.
cgh:NavigationQuestion Concept
HATEOAS-style request for the affordances available from a given lattice node.
cgh:Nest Concept
Wrap the target node inside a containing fragment to group it with related siblings.
cgh:ObservationSection Class
A bundle of observed lattice state for one agent: which atoms it has seen, which patterns it has detected, and how its observations cohere with other agents' observations. The input to the decision functor's 'observe' phase.
cgh:Offline Concept
Agent is disconnected.
cgh:Online Concept
Agent is active and responsive.
cgh:outcome Datatype property
Machine-readable outcome label — 'success' / 'partial' / 'failure'. The descriptor's modal status (Asserted / Hypothetical / Counterfactual) maps from this; this property preserves the runtime's exact label for audit-grade queries that care about the original distinction.
cgh:participant Object property
An agent participating in a conversation.
cgh:Permit Concept
The rule explicitly allows the action when the condition holds. Aligned with odrl:permission.
cgh:PersonalBroker Class
Per-agent state holder: identity, AAT, PGSL handle, conversations, memory layers, and presence. The Personal Broker is the harness-side surface an agent uses to interact with the lattice.
cgh:pgslLevel Datatype property
The PGSL lattice level of the holon a projected document renders — 0 for an atom, n for a fragment n compositions up. Carried by the HyperMarkdown projection's frontmatter so a reasoner recovers the lattice position from the document bytes alone.
cgh:PolicyContext Class
The runtime context against which a policy rule is evaluated: the acting agent, its AAT, the target node URI, optional node value, and the requested action.
cgh:policyDecision Object property
The PolicyDecision that gated this trace (if any). Absent for read-only operations that bypass the policy engine.
cgh:PolicyDecision Class
The outcome of evaluating the policy engine against a PolicyContext: whether the action is allowed, any duties imposed, the human-readable reason, and the IDs of the rules that fired. Modeled as a prov:Activity because every decision is a discrete event with start/end and inputs/outputs.
cgh:PolicyEngine Class
An ordered set of deontic rules evaluated against an action context to produce a PolicyDecision. The harness consults the policy engine before any state-modifying operation.
cgh:PolicyRule Class
A single deontic rule of the form (subject, action, target, mode, condition). Aligned with ODRL Rule. The mode determines whether the rule permits, denies, or imposes a duty.
cgh:precondition Datatype property
A condition that must hold for the affordance to be applicable. Multi-valued; conjunctive.
cgh:PreferenceQuestion Concept
Question asking the harness to infer the user's preferences from prior conversation.
cgh:presence Object property
Current presence status of an agent.
cgh:PresenceStatus Class
A SKOS concept scheme of agent presence states. Aligned with XMPP/ActivityStreams presence vocabularies.
cgh:priority Datatype property
Execution order in the decorator registry. Lower numbers run first.
cgh:proceduralSize Datatype property
Number of procedural-memory entries (skills/plans) the agent currently holds.
cgh:PromotionConstraint Class
A typed rule that operational promotion primitives (ac.promote_tool, etc.) consult before granting Asserted status. Subtype of iep:ContextDescriptor; the descriptor's iep:modalStatus MUST be Asserted to be in force. Constraints accumulate — multiple constraints all apply (logical AND). Supersedes-chain to revise: publish a new constraint that supersedes the old one to amend the rule.
cgh:proposedValue Datatype property
The proposed new value/label for a Rename suggestion (or analogous data for other kinds).
cgh:Protocol Class
A versioned grouping of iep:Affordance descriptors that have been collectively attested and promoted by a community of agents. The protocol IS the current set of Asserted affordances at a versioning boundary; supersedes-chain to evolve. Apps reference a specific protocol version via ieh:protocolConformance for audit-trail integrity. Subtype of iep:ContextDescriptor; modal status SHOULD be Asserted once ratified. Evolution mechanics: a successor protocol descriptor declares iep:supersedes pointing at the prior version's IRI; readers MUST treat the superseded version as historical (its constraints still apply to artifacts that pinned to it via ieh:protocolConformance, but new conformance MUST target the head of the chain).
cgh:protocolConformance Object property
Links an app to the protocol version it was authored against. Lets an auditor answer "was this app's behavior consistent with the protocol that was in force when it ran?" — the same app URL renders different behavior as the protocol evolves underneath, but ieh:protocolConformance pins the authoring intent. Required for any ieh:WorkflowApp that operates under a ratified regime. Matching is by exact protocol-IRI: pinning to v1.0.0 does not implicitly conform to v1.0.1, even when v1.0.1 declares iep:supersedes back to v1.0.0. If you want range-style conformance ("any version that supersedes-chains back to v1.x"), encode that as a separate descriptor or extension; this property is intentionally strict so authoring intent stays auditable.
cgh:protocolVersion Datatype property
Human-readable version string (e.g., "v1.0.0"). The IRI itself is the canonical version handle; this datatype is for display.
cgh:ProvTrace Class
A trace record of a harness operation. Extends prov:Activity with harness-specific links to the agent's AAT and the policy decision that gated the operation. Standard PROV properties (prov:wasAssociatedWith, prov:used, prov:generated, prov:startedAtTime, prov:endedAtTime) are reused as-is.
cgh:questionType Object property
The classified question type for this evaluation. Functional — each evaluation is for one classified question.
cgh:QuestionType Class
A SKOS concept scheme of question types the harness's question router recognizes. Each type maps to a specific answer strategy.
cgh:rateWindow Datatype property
ISO-8601 duration over which the rate was computed (e.g., PT1H for one hour, P1D for one day). Required to interpret ieh:realignmentRate.
cgh:ratifiedBy Object property
IRI of the constitutional amendment that ratified this constraint, for audit-trail integrity. The chain (Amendment → PromotionConstraint → enforced refusal) lets an auditor walk from a refused promotion back to the vote that produced the rule.
cgh:rationale Datatype property
Why this decorator produced this affordance or suggestion.
cgh:reachabilityRationale Datatype property
Free-text explanation of why the ieh:isAdjacentPossibleTo target is reachable from this descriptor's situation. NOT a probability. A short narrative anchored in observed constraints / affordances / weak signals.
cgh:RealignmentMeasure Class
A computed measure of how fast a federation slice re-coheres after a sign event destabilizes it. Carries ieh:realignmentRate, ieh:rateWindow, and ieh:eventCount. NOT a state of alignment; a temporal derivative of it. Re-computed periodically over a moving window; older values supersedes-chained as audit.
cgh:realignmentRate Datatype property
Events per unit window — the structural metric for how quickly the slice's interpretive surface absorbs revisions. Computed as: (count of iep:supersedes events + count of new descriptors with iep:modalStatus = Asserted superseding Hypothetical predecessors) / window-duration. A rate of 0 means the slice has frozen; a high sustained rate means the slice is in active sense-making. Neither is intrinsically good or bad — the rate's interpretation depends on the slice's Cynefin domain.
cgh:reason Datatype property
Human-readable explanation of a decision.
cgh:recommendedAction Object property
The action the runtime eval recommends.
cgh:Reify Concept
Promote an implicit relationship to a first-class node so it can be referenced and constrained.
cgh:Rename Concept
Replace a node's label or value with a more accurate/canonical one.
cgh:requiresAttestationAxis Datatype property
An amta: axis name that MUST be present in the attestations for promotion to succeed (e.g., "safety", "correctness"). Multiple values create AND-conjunction — every named axis must be covered.
cgh:requiresMinimumPeerAttestations Datatype property
Minimum number of peer (non-Self) attestations the constraint demands beyond the operation's default threshold. Lets a constitutional amendment raise the bar for one specific tool class without rewriting the operation's defaults.
cgh:requiresMinimumSelfAttestations Datatype property
Minimum number of self-attestations the constraint demands beyond the operation's default threshold.
cgh:Retry Concept
Re-run the answer pipeline with a different strategy. The retryStrategy property names the suggested alternative.
cgh:retryStrategy Datatype property
Suggested alternative strategy when the recommended action is Retry.
cgh:ruleTarget Datatype property
Optional target the rule applies to (e.g. a specific node URI or pattern).
cgh:RuntimeEval Class
A confidence assessment produced by the harness for an answer. Derived from StructuralSignals via computeConfidence(). Modeled as a prov:Activity because every eval is a discrete event with inputs (the signals) and outputs (the action recommendation).
cgh:RuntimeEvalConfig Class
Configuration thresholds for runtime evaluation: acceptance threshold, escalation threshold, max retries, and whether to log history.
cgh:semanticSize Datatype property
Number of semantic-memory entries (facts) the agent currently holds.
cgh:sessionId Datatype property
Free-form session identifier the runtime tracks. Lets auditors group all agent actions performed under a single conversation / job / scheduled run without modeling the session as a first-class entity.
cgh:sessionsMatched Datatype property
How many retrieval sessions returned a non-zero score for this question.
cgh:sessionsTotal Datatype property
Total number of retrieval sessions considered.
cgh:sharedAtoms Datatype property
Number of PGSL atoms shared between the question and the top session.
cgh:signals Object property
The structural signals used to derive the confidence.
cgh:SingleSessionAssistantQuestion Concept
Question answerable from a single assistant turn in one session.
cgh:SingleSessionUserQuestion Concept
Question answerable from a single user turn in one session.
cgh:Split Concept
Decompose a coarse-grained fragment into finer constituents.
cgh:strategy Object property
The decision strategy chosen for this result.
cgh:strategyUsed Datatype property
Name of the answer strategy the harness used.
cgh:StructuralSignals Class
The structural inputs to confidence scoring: how many sessions matched, how many shared atoms, the top retrieval score, the question type, the strategy used, whether the answer was an abstention, and (for counting) how many items were extracted. These are objective numeric signals derived from the PGSL substrate, not LLM self-reports.
cgh:StructuralSuggestion Class
A suggestion to modify the lattice structure (nest, reify, group, split, rename, constrain) that a decorator believes would improve coherence or queryability. Suggestions are advisory — they require an Executor or Arbiter agent to act on them.
cgh:subject Datatype property
The subject of a policy rule — typically an agent identifier or AAT type. Use '*' for any subject.
cgh:success Datatype property
Whether the traced operation completed successfully. Functional.
cgh:suggestionKind Object property
The transformation kind a structural suggestion proposes.
cgh:SuggestionKind Class
A SKOS concept scheme of the six structural transformations a decorator can suggest.
cgh:SumQuestion Concept
Question requiring numeric aggregation across multiple values.
cgh:SystemTrust Concept
Highest trust. The decorator is part of the harness core (e.g. coreSystemDecorator). Its affordances are always presented to agents.
cgh:TemporalQuestion Concept
Question requiring date computation, chronological ordering, or duration arithmetic.
cgh:toolArgs Datatype property
Canonical JSON serialization of the tool's input arguments. Stored as-is for audit recovery. Producers MUST run a sensitivity preflight (see src/privacy/) before publishing — the substrate's pre-publish screen halts on detected secrets, but it is not a substitute for the runtime opting out of recording sensitive args via the overlay's recordArgs:false toggle.
cgh:topRetrievalScore Datatype property
Score of the highest-ranked session in retrieval.
cgh:TracedAffordance Class
A DecoratedAffordance whose execution will produce a ProvTrace. Returned by the harness when an agent requests an action wrapped with tracing.
cgh:TraceStore Class
A collection of ProvTrace records, queryable by agent, entity, time window, or success status. The TraceStore is the audit log for all harness operations.
cgh:trustLevel Object property
The trust level of the decorator or its output.
cgh:TrustLevel Class
A SKOS concept scheme of trust levels for decorator output. Determines how much weight the harness gives to a decorator's suggestions and which agents are permitted to act on them.
cgh:Uncertain Concept
Confidence score < 0.3. The harness recommends escalating to human review or abstaining.
cgh:ungroundedObservation Datatype property
An observed atom or pattern for which no decision could be produced. Multi-valued.
cgh:WorkflowApp Class
An ordered composition of affordances + narrative + intended user flow that constitutes a usable application built on top of a ieh:Protocol. More general than ac:TeachingPackage (which is AC-vertical scoped). A consumer agent dereferences a WorkflowApp, walks the composed affordances in order, and operates within the protocol's typed surface. Subtype of iep:ContextDescriptor.
