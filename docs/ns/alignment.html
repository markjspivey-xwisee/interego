Interego 1.0 Cross-Layer Alignment Ontology align:

Namespace https://markjspivey-xwisee.github.io/interego/ns/alignment# · version 1.0 · 35 terms.

Cross-layer alignment ontology that ties together the three Interego ontologies (iep:, pgsl:, ieh:) and their external W3C alignments (PROV-O, Hydra, ODRL, ACL, VC, DCAT, OWL-Time). Importing this ontology gives you the full picture: typed context descriptors ABOUT lattice content WRAPPED BY agent harness machinery, all grounded in standard W3C vocabularies. The alignment is declarative — every cross-layer relationship is an OWL axiom annotated with the design rationale.

Machine-readable form: alignment.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. align:affordanceOn = https://markjspivey-xwisee.github.io/interego/ns/alignment#affordanceOn).

Terms

align:affordanceOn Object property
Alias for ieh:affordanceTarget, emphasizing that affordances are directed at a specific PGSL node. Use this when the emphasis is on navigation ('what can I do from HERE?'), ieh:affordanceTarget when the emphasis is on the affordance itself.
align:answersInterrogative Object property
Declares that instances of the subject class can serve as grounding data for responses to the object interrogative type. Used to map lower-layer technical classes to the user-facing ie: interrogative grammar. Multi-valued in both directions: a class can answer multiple interrogatives, and an interrogative can be answered from multiple classes.
align:ArbitratedCompositionPattern Concept
Pattern: two or more Executor agents produce conflicting ContextDescriptors for the same subject → an Arbiter AAT reads both, evaluates coherence → produces a iep:ComposedDescriptor using the override operator, with the Arbiter's policy decision attached as provenance.
align:AuditedPublicationPattern Concept
Pattern: an Archivist AAT promotes a PGSL fragment from local tier to pod tier → the promotion produces a ieh:ProvTrace → the trace references the specific fragment (prov:used) and the new location (prov:generated) → the promotion is only allowed if the policy engine permits it for the Archivist's AAT.
align:confidenceToTrust Object property
Links a runtime evaluation to the iep:TrustFacet it materializes. The harness's numeric confidence (ieh:confidence) projects onto iep:epistemicConfidence, and the ieh:ConfidenceLevel projects onto a iep:TrustLevel via this relation.
align:contextLattice Object property
The PGSL Lattice a DecoratorContext is part of. Functional — a context is always from one lattice.
align:contextNode Object property
The PGSL node at the center of a DecoratorContext. Functional — each context is for exactly one node.
align:deprecatedAt Datatype property
align:DeprecationMarker Class
Attaches to a term descriptor to signal it has been deprecated. Carries the replacement term IRI (if any), the deprecation date, and the version after which the term is removed. Modal status is Counterfactual on the term's claim of being currently-valid.
align:describesAtom Object property
Specialization of iep:describes for individual PGSL atoms. Useful for attaching fine-grained trust / temporal / semiotic facets to specific atoms without wrapping them in a fragment.
align:describesFragment Object property
Specialization of iep:describes for the case where the described graph's content is a PGSL fragment. Declared as a subPropertyOf so any iep:describes query automatically includes fragment-level descriptions.
align:describesLattice Object property
Specialization of iep:describes for an entire pgsl:Lattice instance. Used when a context descriptor applies to ALL content in the lattice (e.g. a pod-wide provenance statement).
align:evaluatedNode Object property
A runtime evaluation references the PGSL node whose retrieval/answer was scored. Subproperty of prov:used so the evaluation's PROV inputs are discoverable.
align:FederatedObservationPattern Concept
Pattern: multiple agents in different pods each produce ObservationSections over the same logical subject → their observations are unioned via iep:union composition → coherence between agents is computed from the intersection of their observed atoms → a Decision is produced only when coherence ≥ threshold.
align:hasMapping Object property
align:IntegrationPattern Class
A SKOS concept scheme of canonical integration patterns across the three layers. Each pattern names a standard way of using iep:, pgsl:, and ieh: together to achieve a common outcome.
align:mappingRelation Object property
owl:equivalentClass | rdfs:subClassOf | skos:closeMatch | skos:relatedMatch | align:Renamed | align:Narrowed | align:Widened
align:NamespaceBridge Class
A first-class descriptor that documents an alignment between two namespaces (or two versions of one namespace). Carries source + target prefixes, the per-term mapping, and a confidence/exact-match flag per pair. Bridges are themselves linked data — versionable, citable, queryable.
align:observedAtom Object property
An ObservationSection references the specific PGSL atoms it observed. Declared as subPropertyOf prov:used so standard PROV tooling can walk from a decision back to the atoms that grounded it.
align:policyDecisionToAccessControl Object property
Links a ieh:PolicyDecision to the iep:AccessControlFacet it authorizes. A granted policy decision materializes as an authorization triple in the AccessControlFacet.
align:policyToAclAuthorization Object property
Projection from a harness PolicyRule to a W3C Web Access Control Authorization. Not an equivalence — ACL is flat (agent × mode × resource), while ODRL/cgh policy rules can carry conditions and obligations.
align:produces Object property
A harness activity (e.g. RuntimeEval, Decision, PolicyDecision) can produce a ContextDescriptor as its output. The produced descriptor then describes the subject entity with whatever facets the harness computed.

align:temporalIntervalOf Object property
Links a iep:TemporalFacet to the underlying OWL-Time ProperInterval. Use this when you need to reason temporally with standard OWL-Time axioms (before, after, during, etc.).
align:TermMapping Class
A single mapping between a source term and a target term, with a relation (owl:equivalentClass / rdfs:subClassOf / skos:closeMatch / skos:relatedMatch). Encoded as a node in the bridge's graph.
align:TracedAnswerPattern Concept
Pattern: agent asks a question → harness runs retrieval over PGSL → runtime eval scores the answer → a ieh:ProvTrace records everything → the trace is materialized as a iep:ProvenanceFacet on the answer's ContextDescriptor → the descriptor is published. Every link from the final answer back to the source atoms is walkable via PROV-O.
align:traceToProvenance Object property
Links a ieh:ProvTrace to the iep:ProvenanceFacet that records it in a context descriptor. Since ProvTrace is already a prov:Activity, this is a re-materialization of the trace as a first-class provenance facet on the affected entity.
align:TrustedRetrievalPattern Concept
Pattern: retrieval returns candidate PGSL atoms → each atom is filtered by the iep:TrustFacet of the ContextDescriptor that describes it → only atoms with trust level ≥ required threshold are passed to the answer pipeline → the runtime eval's confidence is capped by the minimum trust across used atoms.
align:trustFromCredential Object property
A iep:TrustFacet can be backed by a W3C Verifiable Credential. This alignment just restates iep:verifiableCredential with a clearer name for cross-layer use.
align:VersionMigration Class
A specialized bridge between two versions of the SAME namespace (e.g. iep:1.0 → iep:2.0). Carries per-term added / removed / renamed / narrowed / widened classifications so consumers can detect breaking changes.
