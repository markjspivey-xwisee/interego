CTS — Compositional Tuple Store cts:

Namespace https://markjspivey-xwisee.github.io/interego/ns/cts# · 21 terms.

Usage-based linguistic substrate. Records utterances as tuples (positions × fillers × context); infers patterns as statistical regularities; lets interoperability emerge without a pre-specified schema. Underlies PGSL at the substrate layer and iep:ParadigmSet / iep:SyntagmaticPattern at the grammar layer.

Machine-readable form: cts.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. cts:CharacterAtom = https://markjspivey-xwisee.github.io/interego/ns/cts#CharacterAtom).

Terms

cts:CharacterAtom Class
An atomic glyph holon — a single character at the smallest granularity. Identity is content-addressed (urn:pgsl:atom:<sha256-of-character>). Two distinct occurrences of 'm' across all tuples share one CharacterAtom IRI.
cts:Context Class
The surrounding environment of a tuple — time, place, agents present, prior tuples, task being performed. Realized as a iep:ContextDescriptor that accompanies the tuple.
cts:Emergence Class
The phenomenon by which interoperability between agents arises without explicit schema agreement. Two agents using the same tuples converge on compatible patterns; the emergence is the quantitative observation that their patterns align over time. Constructed from sets of cts:Pattern via cross-agent comparison.
cts:Filler Class
A value occupying a position in a tuple. Can be a pgsl:Atom, a iep:ContextDescriptor, a WebID, or any IRI-addressable entity.
cts:hasFiller Object property
cts:hasPosition Object property
cts:inducesPattern Object property
Relates a set of tuples to the pattern they collectively evidence. Inverse of pattern membership.
cts:length Datatype property
Number of constituent atoms / characters in the occurrence. Together with cts:offset uniquely locates the inclusion morphism within the containing tuple.
cts:next Object property
Hydra-style link relation pointing to the immediately-following position in the same tuple. Absent on the last position. Lets an agent walk a tuple's positions without numerical indexing — the same pattern as following a paginated collection.
cts:ngram Object property
The canonical n-gram resource being instantiated by this occurrence. Multiple Occurrences across different containing tuples may share the same cts:ngram value — that is the substructure-sharing relation made explicit.
cts:NGram Class
A canonical, deduplicated sequence of CharacterAtoms (or higher-level fillers). Identity is content-addressed: two tuples containing the same bigram literally share the NGram IRI. The vocabulary of NGrams is the generating set of the BPE-as-colimit construction.
cts:Occurrence Class
A typed morphism recording that a specific NGram appears in a specific containing tuple at a specific offset. Conceptually a hyperedge (NGram, ContainingTuple, offset, length). Querying 'what tokens share substructure with X' is a SPARQL join through this class — substructure sharing is graph topology, not algorithm.
cts:offset Datatype property
Zero-based starting position of the n-gram within the containing tuple.
cts:ParadigmSet Class
The distribution of fillers observed at a given position across all tuples. Used to infer the class of fillers valid at that position (i.e. the paradigm) — cf. Saussure's paradigmatic axis.
cts:Pattern Class
A statistical regularity across tuples — e.g. 'position 1 is almost always filled by a hela:Actor' or 'position 3 and position 5 correlate'. Learned from observed tuples, not declared.
cts:Position Class
A slot in a tuple where a filler may appear. Positions are ordered and typed by the paradigm set of fillers admissible there. A structural coordinate in a pgsl:Fragment.
cts:previous Object property
Inverse of cts:next; absent on the first position.
cts:Tuple Class
A single usage record: a sequence of positions, each filled by a specific filler, observed in a specific context. Realized as a pgsl:Fragment — the atomic unit of the CTS.
cts:usedIn Object property
cts:within Object property
The tuple this occurrence lives inside. Structurally: a parent in the holon hierarchy. The Janus-flip is observable here — the NGram is a whole over its CharacterAtoms AND a part within this containing tuple via the Occurrence.
cts:withinTuple Object property
The tuple this position belongs to. Structurally: the parent in the holon hierarchy. Inverse of cts:hasPosition.
