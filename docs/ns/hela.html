HELA — Topos-theoretic xAPI stack hela:

Models xAPI learning-record data as a presheaf category ℰ = Set^(𝒞_xAPI^op). Provides the categorical structure that lets learning records compose, refine, and sheafify. Aligns with iep:ProvenanceFacet + iep:TemporalFacet + iep:TrustFacet.

Terms

hela:Actor Class
The learner / agent whose experience a statement records. Subclass of prov:Agent so standard provenance tooling navigates the xAPI graph.
hela:LearningObject Class
The object of an xAPI statement — what the actor experienced / did. As a sub-presheaf of hela:Trace.
hela:presheafOf Object property
Declares that a hela:Trace is the presheaf of a particular category (subcategory of 𝒞_xAPI, e.g. course C's statements).
hela:Statement Class
An xAPI-style (actor, verb, object) statement — the atomic unit of the presheaf category 𝒞_xAPI. Every statement is simultaneously a prov:Activity (with attribution + timing) and an object in ℰ (with morphisms to refinements / aggregations).
hela:SubobjectClassifier Class
The Ω of the topos ℰ — classifies which subsets of statements constitute 'valid' or 'complete' views for a given learner / certificate. Constructed at runtime from a pgsl:Fragment (the trace structure) and a iep:SemioticFacet (the validity criterion). See src/pgsl/classifier.ts.
hela:Trace Class
A learner's full sequence of statements over time — a presheaf ℰ : 𝒞_xAPI^op → Set. Realized as a iep:ContextDescriptor whose facets carry the PROV + Temporal structure.
hela:Verb Class
The action type in an xAPI statement (experienced, completed, attempted, passed). Primitive; serves as morphism-label between hela:Statement objects.
