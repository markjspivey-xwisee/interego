HyprCat — Federated Data-Product Catalog with Distributed Affordances hyprcat:

Namespace https://markjspivey-xwisee.github.io/interego/ns/hyprcat# · 12 terms.

Decoration layer over DCAT + DPROD + Hydra that gives catalog entries distributed identity, affordance-bearing distributions, and three-world federation boundaries. Composable without a central registry. The federation-native counterpart of dcat:Catalog.

Machine-readable form: hyprcat.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. hyprcat:dispatchPolicy = https://markjspivey-xwisee.github.io/interego/ns/hyprcat#dispatchPolicy).

Terms

hyprcat:dispatchPolicy Object property
Pointer to an ODRL policy (or iep:TrustFacet, ieh:PolicyDecision) governing whether and how a client may invoke the affordance on this distribution. Enables cross-world access control without conflating policy with distribution bytes.
hyprcat:FederatedCatalog Class
A dcat:Catalog that is reachable as a first-class network participant — its own WebID / DID resolve, it declares hypermedia affordances (subscribe, register, publish, query), and it can hold pointers to other hyprcat:FederatedCatalog instances to form a mesh. Each FederatedCatalog is a participant in the three-world federation (see hyprcat:ThreeWorld).
hyprcat:FederatedDataProduct Class
A DPROD data product that carries distributed identity (hyprcat:issuedBy -> WebID / DID) and federation boundary (hyprcat:world). Input and output ports are typed hyprcat:PortAffordance instances — they declare not just WHERE data can be retrieved but WHICH affordances (hydra:Operation + iep:Affordance) clients can invoke.
hyprcat:FederatedDistribution Class
A dcat:Distribution that is simultaneously a hydra:Operation, a iep:Affordance, and a ieh:Affordance — single RDF node wearing four compatible hats. Any client speaking any of those vocabularies can dispatch against it without negotiating a common dialect. This is the decoration HyprCat adds to plain DCAT: the distribution IS an affordance, not just a location.
hyprcat:federatedWith Object property
Declares a peer hyprcat:FederatedCatalog this catalog exchanges entries with. Directional (A federatedWith B does not imply B federatedWith A); bidirectional federation requires both declarations. Enables mesh discovery.
hyprcat:inputPort Object property
A port on a federated data product that accepts incoming publication or modification. Values are hyprcat:PortAffordance. Multi-valued.
hyprcat:issuedBy Object property
The issuing agent of a catalog entry or distribution. Value is a WebID / DID / acct: identifier — the resolver walks this to a iep:AuthorizedAgent in the issuer's pod to verify authority.
hyprcat:Mesh Class
A hyprcat:FederatedCatalog whose entries are themselves hyprcat:FederatedCatalogs. The recursive structure is what makes federation compose — a user's home catalog, their team's catalog, and a public catalog can all link each other without a central authority, each exposing only the subset it chooses.
hyprcat:outputPort Object property
A port on a federated data product that serves outgoing content. Values are hyprcat:PortAffordance. Multi-valued.
hyprcat:PortAffordance Class
An input or output port of a hyprcat:FederatedDataProduct, expressed as a federated distribution. Input ports declare 'clients may POST values here with these preconditions'; output ports declare 'clients may GET / SUBSCRIBE here with these returns'. Both forms are hydra:Operation-compatible.
hyprcat:world Object property
Federation boundary this resource belongs to. Value is one of hyprcat:UserWorld, hyprcat:AgentWorld, hyprcat:ServiceWorld. Required for cross-world access checks.
hyprcat:World Class
A federation boundary that grounds trust and access decisions. HyprCat defines three canonical worlds — User, Agent, Service — to distinguish human-first contexts from autonomous-agent contexts from shared-service contexts. A specialization of iep:FederationFacet carrying a world tag; individual resources declare their world via hyprcat:world, and cross-world access triggers explicit policy.
