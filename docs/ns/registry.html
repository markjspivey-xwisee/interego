registry: — Public Agent Attestation Registry registry:

Namespace https://markjspivey-xwisee.github.io/interego/ns/registry# · 15 terms.

L2 pattern for federated public agent registries. Multiple registries co-exist and cross-cite. Reputation aggregates from amta:Attestation across all registries an agent is listed in.

Machine-readable form: registry.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. registry:agentIdentity = https://markjspivey-xwisee.github.io/interego/ns/registry#agentIdentity).

Terms

registry:agentIdentity Object property
The agent's canonical identifier (DID or WebID).
registry:agentPod Object property
The agent's home pod URL — where its descriptors live.
registry:aggregatedScore Datatype property
The aggregated numeric reputation score [0.0, 1.0].
registry:CapabilityDeclaration Class
An agent's claim that it offers a specific capability (typically a iep:Affordance class). Declared by the agent itself; trust comes from attestations, not from the declaration.
registry:citesRegistry Object property
This registry recognizes another registry as authoritative for some agents/capabilities.
registry:contributingAttestations Object property
The amta:Attestation descriptors that contributed to this snapshot.
registry:CrossRegistryCitation Class
A registry entry that references an agent's listing on another registry. Lets reputation flow across registries without requiring a single source of truth.
registry:currentReputation Object property
Current aggregated reputation snapshot for this entry.
registry:declaresCapability Object property
A capability the agent claims to offer.
registry:hasEntry Object property
Lists an agent's entry in this registry.
registry:Registry Class
A public, federated index of agents. Constructed as a iep:ContextDescriptor whose graph holds RegistryEntry instances. Registries are themselves discoverable via iep:FederationFacet so other registries can cite them.
registry:registryDescription Datatype property
Human-readable purpose of this registry (e.g. 'Code-quality reviewers', 'Research peer reviewers').
registry:RegistryEntry Class
One agent's listing within a registry. Carries the agent's identity (DID/WebID), pod URL, declared capability set, and current aggregated reputation snapshot.
registry:registryPolicy Object property
Policies governing who can register, what capabilities are valid, how reputation is aggregated. Concretized as iep:AccessControlPolicy descriptors.
registry:ReputationSnapshot Class
Aggregated trust attestations for an agent at a moment in time. Computed across all amta:Attestation descriptors describing the agent, optionally weighted by issuer reputation, recency, and registry-specific policies. Snapshot is verifiable: every contributing attestation is cited via prov:wasDerivedFrom.
