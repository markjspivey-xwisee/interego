OWM — Organizational Working Memory 1.0 Ontology owm:

Namespace https://markjspivey-xwisee.github.io/interego/applications/organizational-working-memory/owm# · version 1.0 · 35 terms.

Vertical vocabulary for an organization's shared working memory: the decisions, organizational policies, projects, follow-up action items, knowledge notes, and people an agent records and later retrieves. Payload terms carried inside Interego (iep:) typed context descriptors — the descriptor layer supplies identity, provenance, temporal validity, and access control; owm: supplies the organizational domain.

Machine-readable form: owm.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. owm:about = https://markjspivey-xwisee.github.io/interego/applications/organizational-working-memory/owm#about).

The decentralized identifier of the agent that authored or is bound to this record.

An alternative name a subject is also known by, for retrieval.

The person who made a decision.
owm:decision Object property
Links a follow-up or note to the decision it derives from.
owm:Decision Class
A recorded organizational decision: what was decided, by whom, and the rationale. The durable memory of a choice so later agents can retrieve the reasoning rather than re-litigate it.
owm:desc Datatype property
A free-text description of the record.
owm:dueAt Datatype property
The due date/time of a follow-up.
owm:followup Object property
Links a decision or note to a follow-up action item it spawned.
owm:FollowUp Class
A follow-up action item captured against a decision, project, or note — an open loop with an optional due date and owner that the working memory tracks to closure.
owm:KnowledgeNote Class
A captured unit of organizational knowledge — an observation, fact, or context worth remembering — attached to the subjects, project, or context it is about.
owm:note Datatype property
The text body of a knowledge note.
owm:objective Object property
An objective a project or decision serves.
owm:organization Datatype property
The name of the organization a record belongs to.
owm:OrgPolicy Class
A standing organizational policy: a typed, bodied rule the organization commits to, which agents consult at the point of work.
owm:participant Object property
A person participating in the project or decision.
owm:person Object property
A person referenced by this record.
owm:Person Class
A person referenced in organizational working memory — a decision maker, participant, watcher, or owner. Identified by a DID/WebID.
owm:policy Object property
An organizational policy this record invokes or is governed by.
owm:policyBody Datatype property
The normative text of an organizational policy.
owm:policyType Datatype property
The category of an organizational policy (e.g. security, conduct, data-handling).
owm:project Object property
The project this record is filed under.
owm:Project Class
A project the organization is running — the unit that decisions, notes, follow-ups, and participants are scoped to.
owm:queuedAt Datatype property
When a follow-up was queued.
owm:rationale Datatype property
The reasoning recorded for a decision.
owm:recordedAt Datatype property
When a decision or record was recorded.
owm:role Datatype property
The role a participant plays on a project or in a decision.
owm:scope Datatype property
The working scope a record is filed under.
owm:status Datatype property
The lifecycle status of a follow-up or decision (e.g. open, closed, superseded).
owm:tag Datatype property
A retrieval tag applied to a record.
owm:upsertedAt Datatype property
When a record was last created or updated.
