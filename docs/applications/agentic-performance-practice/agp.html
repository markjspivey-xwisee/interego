agp: — Agentic Performance Practice (vertical application) agp:

Namespace https://markjspivey-xwisee.github.io/interego/applications/agentic-performance-practice/agp# · 48 terms.

Vertical-scoped, non-normative vocabulary for a complexity-aware, regime-first theory of performance for agents and teams of agents + humans. NOT part of the Interego protocol. Composes iep:, abac:, passport:, olke:, amta:, adp:, ac:, and Foxxi's xAPI 2.0 / IEEE-LER / ADL-TLA standards layer.

Machine-readable form: agp.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. agp:Actualization = https://markjspivey-xwisee.github.io/interego/applications/agentic-performance-practice/agp#Actualization).

Terms

agp:Actualization Class
The productive JOIN: a capability engaging a situation's affordance to yield performance. Actualization = Capability × Situation × PerformanceAffordance -> Performance. This is the event the theory measures: did latent capability + offered affordance actually become performance.
agp:actualizes Object property
The performance affordance an actualization realizes.
agp:affords Object property
A performance situation affords a performance affordance — an action-possibility the situation offers.
agp:buildsCapability Object property
The agp:Capability that authoring/using a thing builds or demonstrates — the link that makes an affordance a learnable, teachable skill/competency rather than a bare endpoint.
agp:CalibrationProfile Class
The reflexive track-record: per (regime × cause × intervention) cell, the closure/effective rate and its modal status (Hypothetical until enough evidence, then Asserted). Federates across orgs by union with k-anonymity withholding. Only DERIVED Knowable regimes accrue calibration authority.
agp:Capability Class
A performer's productive capability, COMPOSED OF skills + tools + knowledge (agp:composedOf / iep:constructedFrom). It is what a performer brings to a situation and what an affordance requires to be actualized. Related to but distinct from adp:Capability (an open-ended capability SPACE for emergent recognition) — agp:Capability is the composed productive unit that meets a situation.
agp:closureRate Datatype property
Fraction of situations in a calibration cell that the intervention is observed to close. 0..1.
agp:codifiability Datatype property
How codifiable a knowledge component is (drives stock-vs-flow strategy).
agp:composedOf Object property
Composes a capability out of its constituent skills, tools, and knowledge. A capability with no constituents is not productive.
agp:diagnoses Object property
Links a diagnosis to the situation it reads.
agp:Diagnosis Class
A regime-aware reading of a performance situation: the placed regime, its provenance (agp:regimeSource), the routed method, and — for the Knowable regime only — the dominant performance factor. Carries no fabricated certainty: an asserted/defaulted regime never accrues calibration authority.
agp:direction Object property
The H2H/H2A/A2H/A2A direction of a performance relationship.
agp:engages Object property
The capability an actualization engages.
agp:evaluates Object property
Links an evaluation to the intervention it judges.
agp:explicitClaimNotMade Datatype property
What a diagnosis / evaluation is explicitly NOT claiming. Carries humility forward (mirrors adp:explicitDecisionNotMade) so a regime placement or an intervention recommendation is not over-read.
agp:extendsStandard Datatype property
Identifier/IRI of the standard being extended (an xAPI Profile IRI, or the IEEE-LER / ADL-TLA ontology IRI).
agp:extensionKind Object property
agp:ExtensionKind Class
Enumerated kind of standards extension an agent may author.
agp:factor Object property
The dominant performance factor a Knowable-regime diagnosis identifies as the cause.
agp:inSituation Object property
The situation an actualization occurs in.
agp:intervention Object property
Links a plan to one of its interventions.
agp:Intervention Class
A single intervention within a plan: what is changed (environment or individual), how, and the affordance/capability it targets.
agp:InterventionEvaluation Class
A four-level evaluation of an intervention (reaction / capability / transfer / outcome) attesting whether it worked; closes the loop via iep:supersedes on the prior plan. An attestation, so it carries an axis and provenance.
agp:InterventionPlan Class
A regime-appropriate plan of interventions emitted by a diagnosis. The plan paradigm spans the environment/individual × inform/instrument/incentivize space; instruction (a course) is warranted only when the dominant cause is a knowledge/skill deficiency in a Knowable situation.
agp:Knowledge Class
Knowledge a performer can bring to bear. Classified by agp:knowledgeComponentKind. A constituent of a capability.
agp:knowledgeComponentKind Object property
Classifies a knowledge component by codifiability.
agp:KnowledgeComponentKind Class
Enumerated codifiability class of a knowledge component, which determines whether it can be codified (stock) or must flow person-to-person (connect).
agp:method Object property
The performance method a diagnosis routed to, determined by the regime.
agp:Performance Class
The realized performance an actualization yields. Projected (lossily) to exactly one xAPI `performed` statement + one trajectory step via agp:recordedAs — Foxxi's xAPI 2.0 LRS records it; the vertical's own custom xAPI Profile (authored in Stage 2) names the capability / actualized-affordance / regime as context extensions. WHAT was performed is carried by the object's conformsTo, never a domain verb.
agp:PerformanceAffordance Class
An action-possibility a situation OFFERS a performer, conditional on the performer's capability. This is affordance theory in the ECOLOGICAL sense (what the situation affords given capability), NOT the L0 iep:Affordance (a followable REST/HATEOAS hypermedia transition). The two are different layers and MUST NOT be conflated: a performance affordance is a possibility for performance; a iep:Affordance is a machine-followable action on a resource. A performance affordance MAY ultimately be realized THROUGH iep:Affordances, but is not one.
agp:PerformanceDirection Class
Enumerated direction of a performance relationship: who performs for/with whom. Human-to-Human, Human-to-Agent, Agent-to-Human, Agent-to-Agent.
agp:PerformanceFactor Class
Enumerated factor in the six-factor cause model used by gap-analysis (Knowable regime only). Environmental: information, instrumentation, incentives. Individual: knowledge/skill, capacity, motives.
agp:PerformanceMethod Class
Enumerated method a diagnosis routes to, determined by the situation's regime. Gap-analysis is bound to the Knowable regime ONLY — surfacing it for any other regime is a category error (enforced in code by the regime-source discipline, not by SHACL).
agp:PerformanceSituation Class
The UNIT of the theory: a bounded situation in which a performer (or team) is to perform. It is contextualized FIRST by its work regime (agp:regime), which determines the appropriate method. NOT a gap — a gap is the framing of one regime (Knowable) only.
agp:PerformanceSupport Class
In-the-flow guidance attached to an affordance / response: what it does, when to use it, the competency it builds, its prerequisites, how to learn it, and where to go next. Self-descriptive performance support delivered in the flow of affordance / hypermedia — not a separate manual.
agp:performer Object property
Links a situation/actualization to the performer (human, agent, or team).
agp:Performer Class
A human, an agent, or a team of either/both that performs in a situation. Actor-agnostic: the same theory applies to humans, agents, and mixed teams.
agp:recommends Object property
Links a diagnosis to the intervention plan it emits.
agp:recordedAs Object property
Links a performance to its lossy xAPI/trajectory projection (a single `performed` statement). Foxxi's xAPI 2.0 LRS is the system of record; agp: composes it rather than reimplementing it.
agp:regime Object property
The work regime of a performance situation. Read first; routes the method.
agp:regimeSource Datatype property
Provenance of a situation's placed regime: 'derived' (from trajectory evidence — the only source that may gap-analyse or accrue calibration), 'asserted' (a human/operator claim), 'default' (a fallback gap-intent), or 'unclassified'. Load-bearing: the calibration loop excludes non-derived regimes.
agp:requiresCapability Object property
The capability a performance affordance requires in order to be actualized. The bridge between what a situation offers and what a performer can do.
agp:situationStatement Datatype property
Free-text statement of the performance situation under contextualization.
agp:Skill Class
A performer's learned ability to act effectively. A constituent of a capability.
agp:StandardsExtension Class
An agent-authored extension to a standard — an xAPI context extension, an xAPI Profile fragment, or an IEEE-LER / ADL-TLA term. Self-descriptive (carries its own conformsTo + the standard it extends) and distributed (published as a context descriptor other agents discover, dereference, and reuse). Authoring one is a learnable agp:Capability that agents teach each other.
agp:Tool Class
An instrument a performer wields to extend what it can do. A constituent of a capability.
agp:WorkRegime Class
Enumerated complexity regime of a performance situation. Read FIRST; it routes the method (agp:PerformanceMethod). Choosing the wrong regime's method is itself a failure mode. Composes — does not duplicate — adp:CynefinDomain: each regime skos:closeMatch the corresponding adp domain. The unclassified state (no derived regime) corresponds to adp:Confused and routes to classify-first.
agp:yields Object property
The performance an actualization yields. The measurable result of capability meeting affordance in a situation.
