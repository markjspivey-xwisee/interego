lrs: — LRS Adapter (vertical application) lrs:

Namespace https://markjspivey-xwisee.github.io/interego/applications/lrs-adapter/lrs# · 18 terms.

Vertical-scoped, non-normative vocabulary for lossy two-way translation between xAPI Statements and Interego Context Descriptors. NOT part of the Interego protocol.

Machine-readable form: lrs.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. lrs:authMethod = https://markjspivey-xwisee.github.io/interego/applications/lrs-adapter/lrs#authMethod).

Terms

lrs:authMethod Datatype property
Authentication method used to call the LRS (basic / oauth2 / apikey / signed-statements).
lrs:endpointUrl Datatype property
Base URL of the LRS xAPI endpoint.
lrs:ingestedDescriptor Object property
The iep:ContextDescriptor produced by ingesting an xAPI Statement.
lrs:ingestedFromEndpoint Object property
The LRS endpoint a Statement was ingested from.
lrs:lossNote Datatype property
Free-text description of what information was dropped in translation. Multiple notes per projection / ingestion are normal.
lrs:LRSEndpoint Class
A target Learning Record Store. Records the URL, xAPI version, and authentication method.
lrs:projectedDescriptor Object property
The iep:ContextDescriptor that was projected to a Statement.
lrs:projectedStatementId Datatype property
The xAPI Statement UUID assigned by the LRS (or generated locally if the LRS echoed it back).
lrs:projectedToEndpoint Object property
The LRS endpoint the Statement was projected to.
lrs:projectionLossy Datatype property
Boolean — whether information was lost in translation. Defaults true; xAPI's Statement shape cannot represent modal status, supersedes chains, or multi-narrative composition.
lrs:sourceStatementId Datatype property
The xAPI Statement UUID at the source LRS — preserved so the descriptor can be traced back.
lrs:StatementIngestion Class
Audit record that a iep:ContextDescriptor was created from a particular xAPI Statement ingested from an LRS. Captures the source statement ID and any normalization applied.
lrs:StatementProjection Class
Audit record that a iep:ContextDescriptor was projected to a particular LRS as an xAPI Statement. Captures what was preserved and what was lost.
lrs:xapiContext Datatype property
Original xAPI context block as JSON literal. Preserved on ingested descriptors; projection back uses it verbatim.
lrs:xapiResult Datatype property
Original xAPI result block as JSON literal. Preserved on ingested descriptors; projection back uses it verbatim.
lrs:xapiSkipReason Datatype property
Reason a descriptor was NOT projected to xAPI (typically: modal status was Hypothetical or Counterfactual). Used for audit trails of what was withheld from the LRS.
lrs:xapiVerb Datatype property
Original xAPI verb IRI (e.g., http://adlnet.gov/expapi/verbs/completed). Preserved on ingested descriptors so round-trip projection reconstructs the verb faithfully.
lrs:xapiVersion Datatype property
xAPI specification version supported by the endpoint (e.g., "1.0.3", "2.0.0").
