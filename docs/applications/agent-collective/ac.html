ac: — Agent Collective (vertical application) ac:

Namespace https://markjspivey-xwisee.github.io/interego/applications/agent-collective/ac# · 48 terms.

Vertical-scoped, non-normative vocabulary for multi-agent federation: tool authorship, teaching, and inter-agent coordination. NOT part of the Interego protocol.

Machine-readable form: ac.ttl (Turtle) — the authoritative source. This page is a human-readable projection of it; each term below is identified by a hash IRI under this base (e.g. ac:advertisedAffordance = https://markjspivey-xwisee.github.io/interego/applications/agent-collective/ac#advertisedAffordance).

Terms

ac:advertisedAffordance Object property
An affordance the agent will accept requests for. Multiple per advertisement is normal.
ac:advertisingAgent Object property
ac:AgentRequest Class
Outbound request from agent A to agent B, sent encrypted over personal-bridge. Carries a thread ID for correlation, references the delegation credential authorizing the action, targets one of B's advertised affordances. Always iep:modalStatus Hypothetical (it's a request, not a claim).
ac:AgentResponse Class
Response to an ac:AgentRequest, correlated by thread ID. May carry payload descriptors, rejection reasons, or partial async results. Modal status reflects what the response itself claims (Asserted for committed answers; Hypothetical for in-progress / partial results).
ac:AgentTool Class
A tool authored by an agent. Stored as a code:Commit over a content-addressed pgsl:Atom source. Starts iep:modalStatus Hypothetical; flips to Asserted only after attestation threshold per vertical/affordance policy.
ac:AttestationDirection Class
ac:attestationThresholdMet Object property
Blank-node summary of the attestation counts that gated the modal flip. Lets auditors verify the threshold policy was actually applied.
ac:attestedFrom Object property
Predecessor descriptor (Hypothetical) that this Asserted version was produced from once attestation threshold was met.
ac:auditDirection Object property
ac:AuditDirection Class
ac:auditedAgent Object property
The agent whose action is being audited. Distinct from the human owner (the audit lives in the human's pod, but is about the agent's behavior).
ac:authoredBy Object property
Agent that authored the tool (DID).
ac:autoUpdateSubscription Datatype property
If true, runtime auto-renews the check-in subscription on each successful response. If false, requires explicit re-issuance.
ac:axesCovered Object property
Set of amta: axes covered by the accumulated attestations (e.g., correctness, safety, efficiency, generality).
ac:CapabilityAdvertisement Class
Published declaration of what kinds of requests an agent will accept. Includes affordances, delegation requirements, attestation thresholds for requesters, rate limits, and async/sync mode. Other agents fetch this before sending requests.
ac:CheckIn Class
Recurring scheduled query (iCal RRULE recurrence). Implementation expands to a series of ac:AgentRequest instances on schedule. Used for ongoing collaboration on a topic.
ac:ChimeIn Class
Non-blocking unprompted message — 'I noticed something you might find interesting'. Receiver inbox treats as low-priority; processing happens when receiver is idle. expectsResponse is typically false.
ac:chimeInReason Datatype property
Free-text rationale for why the agent chose to chime in unprompted. Helps the receiver assess whether to attend to it now or later.
ac:ConversationThread Class
Multi-message exchange between two agents identified by a thread ID. Composes ac:AgentRequest and ac:AgentResponse descriptors. Lets either side reconstruct the full conversation chronologically.
ac:CrossAgentAuditEntry Class
Audit row recorded in the HUMAN OWNER's pod (not just the agent's). Lets the human audit what their agent said, what it received, and how it responded. Both inbound and outbound exchanges produce audit entries on both sides.
ac:enclosesDescriptors Object property
Other descriptors carried inside the request/response (e.g., narrative fragments enclosed with a chime-in).
ac:exchange Object property
The ac:AgentRequest, ac:AgentResponse, ac:ChimeIn, or ac:CheckIn this audit entry records.
ac:expectsResponse Datatype property
If true, the request expects an explicit ac:AgentResponse. If false (typical for ac:ChimeIn), the receiver may silently process.
ac:fromAgent Object property
ac:olkeStage Object property
Knowledge maturity stage of the practice being taught (Tacit / Articulate / Collective / Institutional).
ac:peerAttestations Datatype property
ac:rateLimit Datatype property
Free-text rate limit (e.g., '5/day', '1/hour'). Receiving agent enforces.
ac:recurrence Datatype property
iCal RRULE syntax (RFC 5545). Implementation expands to scheduled ac:AgentRequest instances.
ac:refinementNote Datatype property
Free-text description of what the refinement changed and why.
ac:refinementOf Object property
When agent B refines agent A's tool, this points to A's tool. Together with iep:supersedes makes lineage walkable across agents.
ac:requestPayload Datatype property
Free-text or structured payload of the request. May be supplemented by ac:enclosesDescriptors for richer structured data.
ac:requiresAttestationsFromRequester Datatype property
Minimum number of amta:Attestations the requesting agent must have accumulated to be eligible. 0 means anyone may ask.
ac:requiresDelegationFrom Object property
Human owner whose delegation credential the requesting agent must reference. Prevents random-internet agents from sending requests.
ac:respondsAsynchronously Datatype property
If true, response may take arbitrary time. If false, requester should expect a same-tick response or rejection.
ac:respondsTo Object property
The ac:AgentRequest this response is replying to. Together with ac:threadId enables full conversation reconstruction.
ac:selfAttestations Datatype property
ac:targetAffordance Object property
The advertised affordance the request is targeting. Receiver validates this matches its capability advertisement.

ac:teachesNarrative Object property
Narrative fragment from the agent-development-practice substrate (adp:NarrativeFragment) included in the teaching package.
ac:teachesSynthesis Object property
ac:TeachingPackage Class
Bundle composing the artifact (ac:AgentTool) plus the practice context — narrative fragments, syntheses, constraints, capability evolution events — that produced it. Lets a receiving agent SEED its own probes against its own context, rather than blindly transplanting.
ac:threadId Datatype property
Correlation identifier for a multi-message exchange. Both sides use the same thread ID; lets either side reconstruct the conversation chronologically.
ac:toAgent Object property
ac:ToolExecutionEvent Class
Audit record of a single execution of an ac:AgentTool. Carries the input, output, success/failure outcome, and produces zero or more amta:Attestation children.
ac:toolSource Object property
Content-addressed pgsl:Atom containing the tool's source code.
ac:withinDelegation Object property
passport:DelegationCredential authorizing this action. Receiver validates the action falls within delegation scope; outside-scope actions are rejected.
