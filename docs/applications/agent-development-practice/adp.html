Terms

adp:amplificationTrigger Datatype property
Narrative pattern that, if observed, increases this probe variant's deployment. Stated up-front so amplification is not retconned.
adp:amplifyProbe Object property
Probe whose deployment is being increased by this evolution step. Provisional.
adp:appliesTo Object property
Links a constraint to the capability it bounds.
adp:basedOnSynthesis Object property
Links an evolution step or constraint to the synthesis it was informed by.
adp:boundary Datatype property
Free-text description of the boundary the agent must operate within. Observable + enforceable; does not specify HOW the agent achieves it.
adp:capability Object property
Links a capability-evolution event or rubric criterion to the capability space it concerns.
adp:Capability Class
An open-ended capability SPACE — not a target threshold. Mastery is recognized through emerging behavior patterns in narrative observations, not threshold-crossing on metrics. The Cynefin domain (adp:cynefinDomain) determines which governance pattern is appropriate.
adp:CapabilityEvolution Class
Recognition of an emergent capability pattern in an agent's behavior. NOT a promotion or certification. Carries explicit-decision-not-made clauses that travel with the agent across deployments, so receiving organizations see what was AND was not claimed.
adp:coherentNarrative Datatype property
An equally-coherent reading of the synthesis. Multiple per synthesis is the norm; the synthesis preserves them rather than picking one.
adp:constraint Object property
Links a capability or probe to the constraints it operates within.
adp:Constraint Class
Governance via boundary, not via prescription. A constraint says 'the agent must not X' or 'must operate within Y'. Observable + enforceable; does not specify HOW the agent achieves the boundary. Refined from synthesis cycles, not declared up-front.
adp:contextSignifier Datatype property
A SenseMaker-style descriptive tag for the situation the agent encountered. Multiple per fragment.
adp:cynefinDomain Object property
Identifies which Cynefin domain a capability space, probe, or situation lives in. Determines which governance pattern is appropriate.
adp:CynefinDomain Class
Enumerated value identifying which Cynefin domain a situation lives in. Determines the governance pattern. Per Snowden, choosing the wrong pattern for the wrong domain is itself a failure mode.
adp:dampeningTrigger Datatype property
Narrative pattern that, if observed, decreases this probe variant's deployment. Stated up-front so dampening is not retconned.
adp:dampenProbe Object property
Probe whose deployment is being decreased by this evolution step. Provisional.
adp:emergedFrom Object property
Links a constraint or capability-evolution event to the synthesis cycles that produced it.
adp:emergentPattern Datatype property
Description of a pattern surfaced by the synthesis. Hypothetical; preserved alongside alternative coherent narratives, not declared as root cause.
adp:emergentSignifier Datatype property
A SenseMaker-style descriptive tag for what emerged from the agent's response. Used to find patterns across fragments.
adp:EvolutionStep Class
Operator decision to amplify some probes and dampen others, based on a synthesis. Carries an explicit-decision-not-made statement to prevent the decision from being read as a 'fix' or 'answer'. The decision is provisional and re-examinable; carries a next-revisit time.
adp:evolutionType Object property
Classifies a capability-evolution event.
adp:EvolutionType Class
Enumerated value classifying a capability-evolution event.
adp:exitsConstraint Datatype property
Free-text description of conditions under which the constraint is relaxed (e.g., user explicitly waives the requirement). Constraints are not absolute; they have explicit exits.
adp:explicitDecisionNotMade Datatype property
Free-text statement of what the operator is NOT claiming with this decision. Counter-cultural and intentional — forces the operator to write down what is being left open, so future readers (humans or agents) do not read amplification as 'fix' or recognition as 'certification'. Travels with the agent across deployments.
adp:fragmentsConsidered Object property
Links a synthesis to the narrative fragments it composes.
adp:hypothesis Datatype property
Free-text statement of what the probe is exploring. Always Hypothetical — explicitly NOT a claim about cause-effect.
adp:NarrativeFragment Class
A qualitative observation: situation signifiers + agent response + emergent signifiers. ALWAYS recorded with iep:modalStatus Hypothetical — we are recording an observation, not asserting causation. Multiple fragments compose into a synthesis.
adp:nextRevisitAt Datatype property
Time at which an evolution decision must be re-examined. Encodes the provisional nature of every amplification / dampening choice.
adp:olkeStage Object property
Knowledge maturity stage from olke: (Tacit / Articulate / Collective / Institutional). A capability-evolution event records what stage the practice has reached.
adp:probe Object property
Links a narrative fragment or synthesis to the adp:Probe whose hypothesis it is observing.
adp:Probe Class
A small parallel experiment. Carries a Hypothetical hypothesis, a variant (prompt / tools / model), explicit amplification + dampening triggers, and a time-bound. Many probes run concurrently; many fail; a few amplify. Never has iep:modalStatus Asserted.
adp:response Datatype property
Narrative description of what the agent did and what followed. Qualitative; not a score.
adp:rubricCriterion Object property
Links a capability to the rubric criteria that guide attention during sensemaking. Multiple per capability.
adp:RubricCriterion Class
A guide for what we care about — NOT a pass/fail threshold. Used during sensemaking to direct attention; never used as a binary acceptance test.
adp:Signifier Class
Short tag emerging from a narrative — used later to find patterns across many fragments. A signifier is descriptive, not evaluative; it labels what is present, not whether it is good or bad.
adp:Synthesis Class
Composed view of many narrative fragments via iep:union. Surfaces recurring signifier patterns; PRESERVES MULTIPLE COHERENT NARRATIVES rather than declaring one root cause. iep:modalStatus stays Hypothetical.
adp:timeBound Datatype property
When to revisit the probe regardless of triggers. Prevents indefinite probes.
adp:variant Datatype property
Identifies the concrete variant being probed (prompt scaffold version, tool inventory, model class, etc.).
